Features and direction
- Smart calendar application planning and build workspace.
- ChatGPT-assisted product and engineering workflow.
- Unified calendar layers for work, personal, public, and conferences.
- Mobile-first dark interface with month, week, and agenda views.
- Smart source scanning with editable criteria and conference discovery focus.
- Manual event creation flow for work, personal, public, and conference layers.
- Weekly automated scan status visible from the settings page.
- Optional lightweight password gate for shared public access.
- Review-first workflow for scanned events before adding them to your own calendar.
- Scan status now includes progress feedback instead of only a final completion state.
- Event creation now feels closer to a real calendar app because it starts from the selected date.
- Repeat scans should now report zero new events when nothing new is discovered.
- Manual scan actions now return structured run details for the frontend to display properly.
- Source management now supports both URL-based ICS feeds and uploaded ICS files.
- Events can now be removed directly from the UI without manual database cleanup.
- Source entries can now be managed with deletion, not just creation.
- Saved events now support two different actions: remove from saved, or delete the event itself.
- Scanned suggestions can now be dismissed if they are not relevant.
- Calendar export test coverage is now isolated from local `.env` password settings.
